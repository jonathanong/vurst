## 2024-11-20 - Sequential Regex Replacement Pipeline Zero-Allocation Optimization
**Learning:** In Rust, when applying multiple sequential text transformations (like regex replacements), passing `&str` and returning `String` causes unnecessary intermediate string allocations on every pass, even if the regex doesn't match and no transformation is needed.
**Action:** Design pipeline functions to accept and return `std::borrow::Cow<'_, str>` instead. This allows the unmodified string (`Cow::Borrowed`) to pass through the entire pipeline with zero intermediate allocations, creating a highly efficient 'happy path'. Delay calling `.into_owned()` until the final step.
