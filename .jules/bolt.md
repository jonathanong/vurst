## 2025-06-20 - Fast-pathing regex replaces to avoid allocation and scanning
**Learning:** In Rust, `regex::Regex::replace_all` incurs significant overhead even when it finds no matches. Attempting to conditionally reassign a `Cow` target using a match block can cause E0505 borrow checker errors.
**Action:** Guard expensive regex executions with highly-optimized `str::contains()` fast-path checks for structural characters (e.g., `![` for image regexes). To satisfy the borrow checker and avoid unnecessary intermediate allocations, mutate a `String` target and update it using `.into_owned()` only when replacements occur.
