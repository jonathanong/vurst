## 2024-05-19 - Reduce allocations in decode_html_entities_once
**Learning:** `regex::Regex::replace_all` can accept closures returning `Cow<'static, str>`. By changing the closure return type from `String` to `Cow<'static, str>`, we can return `Cow::Borrowed` when mapping statically known named HTML entities (if there's no dynamic string concatenation required), significantly reducing temporary string allocations during prompt injection entity normalization.
**Action:** Always check if a regex replacement closure can return a `Cow` (using `Cow::Borrowed` for unmodified substrings or static mapping results) instead of always returning a `String`, especially in high-throughput parsing or sanitization pipelines.
