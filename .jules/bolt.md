## 2024-05-24 - Fast-forwarding string copies over char decoding
**Learning:** In Rust text sanitizers and parsers that split strings primarily by ASCII characters (like `<`), using byte-wise searching (`bytes.iter().position()`) combined with unsafe un-checked UTF-8 conversions (`std::str::from_utf8_unchecked`) to directly append large slices is vastly faster than character-by-character decoding using `.chars().next()`. This avoids the substantial overhead of continuous UTF-8 boundaries checking.
**Action:** Always prefer vectorized byte searching (like `memchr` via `.position()`) to find boundaries when iterating through strings that primarily contain raw text, copying chunks of the valid UTF-8 rather than validating byte-by-byte.
