## 2024-05-24 - Cow vs fast-path mutation
**Learning:** When chaining multiple `Regex::replace_all` calls that return `Cow<str>`, matching and re-assigning back to a single `Cow` variable introduces strict lifetime constraints and borrow checker errors (`E0505`).
**Action:** Instead of complex `Cow` variable shadowing, convert the initial source to a `String`, and use fast-path `str::contains()` checks to completely skip regex evaluation. If the fast-path passes, conditionally mutate the `String` in-place (`if let Cow::Owned(s) = REGEX.replace_all(&current, "...") { current = s; }`). This effectively avoids allocations when there are no matches while satisfying the borrow checker cleanly.
