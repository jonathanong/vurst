## 2024-05-29 - Optimizing URL Sanitization in Rust N-API Modules
**Learning:** Checking for protocol-relative paths inside a Rust `.chars()` iterator consumes unnecessary overhead due to UTF-8 decoding. Iterating over URLs by extracting bytes using `bytes()` avoids full character decoding for checks like `is_ascii_whitespace()` and `is_ascii_control()`. Furthermore, utilizing `matches!(clean_url.as_bytes()[..2], [b'/' | b'\\', b'/' | b'\\'])` allows matching any combinations of forward and backslashes simultaneously, without doing multiple slow `starts_with` calls on strings. In realistic use-cases where URLs generally lack ASCII control characters/whitespace, the performance is >2.5x faster (from ~250ms to ~90ms per million calls) by checking the dirty state using `.any()` before conditionally allocating a clean URL.
**Action:** Always prefer `str.bytes()` iteration instead of `str.chars()` for ASCII-only validations. Use conditional zero-cost allocation (`Option` or an uninitialized variable to store the owned state and pass references) where 99% of paths are cleanly preserved. Combine multiple `starts_with` checks into single byte-slice pattern matches for faster early-return safety guarantees.
