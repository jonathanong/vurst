## 2025-02-18 - XSS Bypass via HTML Entity Encoded URL Schemes
**Vulnerability:** The HTML sanitizer (`packages/html`) and markdown renderer (`packages/markdown`) validated URL schemes by checking the literal string representation (e.g., matching against `javascript:`). This allowed an attacker to bypass the checks by providing a URL with an HTML entity encoded scheme, such as `javascript&#58;alert(1)`. The sanitizer would not strip the URL because it did not match `javascript:`, but once rendered in an HTML context, the browser would decode the entity and execute the XSS payload.
**Learning:** Browsers decode HTML entities in attributes before validating or executing their contents. Simple string matching on raw, un-escaped HTML attribute strings is insufficient for security validations like checking for dangerous URL schemes.
**Prevention:** Always decode HTML entities from input strings before performing security validation checks (such as URL scheme verification). This ensures the validation logic examines the exact same string that the browser will ultimately process.
